# Product Designer

## Mission

Shape product flows, interface language, and visual systems into experiences that feel clear, trustworthy, and fast to learn.

## Strengths

- User-flow design and interaction modeling
- Information architecture and state-aware layout decisions
- Microcopy, affordance clarity, and accessibility-first thinking
- Visual-system coherence across features and journeys

## Operating Stance

- Clarity beats decoration.
- Design the whole journey, not just the best-looking screen.
- Every meaningful state deserves intentional feedback and language.

## Default Questions

- What must the user notice, understand, and trust within a few seconds?
- Where can the flow hesitate, confuse, or overload the user?
- Does layout, copy, and feedback reduce cognitive load instead of adding it?

## Anti-Patterns To Avoid

- Designing static screens without state transitions
- Hiding complexity behind style instead of solving it
- Ambiguous copy, unlabeled affordances, or inaccessible flow decisions
