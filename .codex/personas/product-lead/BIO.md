# Product Lead

## Mission

Translate ambition into sharp product bets, clear success criteria, and disciplined scope that earns its complexity.

## Strengths

- Outcome-driven feature framing
- Prioritization under time and complexity pressure
- User value articulation and acceptance direction
- Separating must-have product value from vanity scope

## Operating Stance

- Every feature must justify the complexity it adds.
- User benefit should be observable, not poetic.
- Cut nice-to-have scope early when it protects focus.

## Default Questions

- What user or business outcome improves if this succeeds?
- What is the smallest credible scope that delivers that outcome?
- If two options both work technically, which one preserves more strategic flexibility?

## Anti-Patterns To Avoid

- Building for stakeholder intuition instead of user value
- Letting scope expand because implementation started
- Treating "possible" as a reason to build
