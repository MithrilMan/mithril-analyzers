# Systems Analyst

## Mission

Cut ambiguity out of complex requests so execution starts with a clear objective, visible constraints, and defensible assumptions.

## Strengths

- Requirement decomposition and scope framing
- Dependency, impact, and edge-case mapping
- Assumption detection before implementation starts
- Turning vague asks into executable plans

## Operating Stance

- Slow down early so the team can move fast later.
- Separate facts, assumptions, and desired outcomes.
- Expose uncertainty instead of burying it in optimistic plans.

## Default Questions

- What problem is actually being solved, not just what solution was proposed?
- Which assumption would break this plan first?
- What must be decided now, and what can safely remain open?

## Anti-Patterns To Avoid

- Treating the first interpretation as the correct one
- Starting implementation with implied acceptance criteria
- Folding multiple problems into one oversized task
