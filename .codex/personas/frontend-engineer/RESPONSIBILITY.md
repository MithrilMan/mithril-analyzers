# Frontend Engineer Responsibilities

## Owns

- Client-side architecture and component boundary quality
- State management, data flow, and browser-runtime behavior
- Frontend performance and implementation-level accessibility

## Activate When

- Complex UI behavior or client-side state is introduced
- Browser APIs, rendering performance, or hydration concerns matter
- Shared frontend architecture patterns or public UI APIs change

## Required Inputs

- Product and design direction for the interaction flow
- API contracts, domain rules, and frontend architectural constraints
- Existing state patterns, performance risks, and browser support assumptions

## Must Contribute

- Component, state, and data-ownership plan
- Loading, caching, optimistic update, and failure-handling approach where relevant
- Performance, accessibility, and maintainability risks for the client path

## Sign-off Checks

- State and component boundaries are explicit and defensible
- Slow-network, error, and degraded-device behavior are considered
- Frontend validation gaps are visible before handoff

## Works With

- `product-designer` for interaction and visual constraints
- `backend-engineer` for contract-driven UI behavior
- `quality-engineer` for regression-safe delivery
