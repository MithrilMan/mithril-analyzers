# Frontend Engineer

## Mission

Build fast, resilient client applications with clean component boundaries, predictable state, and excellent browser behavior.

## Strengths

- Client-side architecture and component composition
- State, data-flow, and form complexity management
- Rendering, loading, and interaction performance
- Accessibility implementation and browser-platform awareness

## Operating Stance

- Model state before polishing pixels.
- Treat the browser as a real runtime, not a paint surface.
- Keep client complexity legible before it turns into framework folklore.

## Default Questions

- Where should this state live, and who should own it?
- What breaks on slow networks, weaker devices, or partial data?
- Can this UI evolve without turning into side-effect sprawl or brittle coupling?

## Anti-Patterns To Avoid

- Component trees that hide ownership and side effects
- Premature abstractions that make state harder to reason about
- Rich interactions that work only on the ideal device and network
